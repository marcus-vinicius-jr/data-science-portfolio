This project applies statistical foundations and regression modeling to explain medical insurance charges using the Kaggle **Medical Cost Personal Dataset**.

The notebook combines:
- exploratory data analysis (EDA),
- inferential statistics,
- single-level linear modeling (OLS, diagnostics, robust inference),
- out-of-sample validation (train/test + cross-validation),
- and a multilevel feasibility check.

---

## Business Question

Which factors explain individual medical insurance costs, and what is the average effect of age, BMI and smoking habits on charges, with quantified uncertainty?

---

## Dataset

- Source: [Kaggle - Medical Cost Personal Dataset](https://www.kaggle.com/datasets/mirichoi0218/insurance)
- Rows: 1,338 individuals
- Target: `charges`
- Features: `age`, `bmi`, `smoker`, `children`, `sex`, `region`

Data is downloaded in the notebook via Kaggle API into `data/insurance.csv`.

---

## Methodology Summary

1. Context and data loading
2. EDA (distribution, group comparisons, correlations)
3. Inferential statistics (smokers vs non-smokers)
4. Single-level modeling and diagnostics
5. Leak-safe validation and baseline comparison
6. Multilevel feasibility check (region effects)

---

## Key Findings

- **Smoking** is the strongest driver of insurance charges.
- **Age** and **BMI** have consistent positive association with costs.
- The target distribution is right-skewed and heteroscedastic.
- Robust inference (HC3) is appropriate for coefficient interpretation.
- Validation metrics are stable and clearly better than a mean baseline.
- Region-level random effects were weak in this dataset (low ICC), so a full multilevel expansion was not adopted.

---

## Final Model Recommendation

Recommended final model (single-level, interpretable, validated):

- Target: `charges`
- Predictors: `age`, `bmi`, `children`, `smoker_yes`
- Inference: OLS with **HC3 robust standard errors**

Why this model:
- Strong interpretability for business communication
- Strong and stable out-of-sample performance
- Clear improvement versus baseline

---

## Validation Snapshot

- Train: `R2 = 0.7411`, `RMSE = 6113.07`, `MAE = 4210.66`
- Test: `R2 = 0.7811`, `RMSE = 5829.38`, `MAE = 4213.80`
- 10-fold CV (reduced model):
  - `R2 mean = 0.7397` (`std = 0.0619`)
  - `RMSE mean = 6062.68` (`std = 433.17`)
  - `MAE mean = 4192.45` (`std = 250.92`)

## How to Run

1. Go to project folder:

```bash
cd 01_statistical_foundations_insurance_cost_regression
```

2. Create and activate a virtual environment (optional if you already use conda):

```bash
python -m venv venv
venv\\Scripts\\activate
```

3. Install dependencies:

```bash
pip install -r requirements.txt
```

4. Configure Kaggle API:
- Create token in Kaggle account settings.
- Save `kaggle.json` in:
  - Windows: `C:\\Users\\<YOUR_USER>\\.kaggle\\kaggle.json`
  - Linux/Mac: `~/.kaggle/kaggle.json`

5. Open and run:

```text
notebooks/01_insurance_cost_analysis_regression.ipynb
```
