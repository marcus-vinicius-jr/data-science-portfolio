# Evaluation of Generative AI Models in Public Health - Malaria Case Study

> **MBA Capstone - USP/Esalq (Data Science & Analytics, 2025)**  
> Part of [data-science-portfolio](../README.md) - Previous Academic Work section.

This project evaluates the quality of responses generated by 10 LLM-based systems
on malaria-related questions, using official WHO (World Health Organization) fact
sheets as the reference source. The study proposes a reusable methodology for
auditing generative AI outputs in sensitive public health contexts.

## Methodology

| Step | Method |
|---|---|
| Readability | Flesch Reading Ease; Flesch-Kincaid Grade Level |
| Text similarity | Cosine Similarity (TF-IDF); Levenshtein Distance; Jaccard Coefficient |
| Composite ranking | Min-Max normalization across all metrics |
| Clustering | K-Means with Z-score standardization; k selected via Elbow and Silhouette |
| Correspondence analysis | MCA with static 2D map and interactive 3D map |
| Statistical validation | One-way ANOVA; Chi-square test |

---

## Key Findings

- Three statistically distinct response clusters were identified, supporting the
  existence of measurable behavioral patterns across AI systems.
- Question topic strongly influenced segmentation, suggesting that prompt content
  can shape response strategy as much as the model selected.
- ScholarGPT and ChatGPT 4.0 achieved the strongest composite performance in this
  study, combining readability, semantic similarity and lexical alignment.
- The proposed methodology can be reused to audit generative AI outputs in health,
  regulatory and other high-sensitivity domains.

## Setup & Execution

Install the required Python packages:

```bash
pip install -r requirements.txt
```

Run the main script from this project folder:

```bash
python code/TCC_MBA_DSA_USP_MALARIA.py
```

The repository already includes precomputed figures and result tables for portfolio
review. The script can regenerate the interactive MCA 3D chart at
`figures/16_mca_perceptual_map_3d.html`; several static exports remain commented in
the original implementation to preserve the capstone workflow.
